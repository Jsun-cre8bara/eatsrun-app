# 핸드폰에서 웹 앱 실행 가이드

## 필수 조건

1. ✅ 컴퓨터와 핸드폰이 **같은 Wi-Fi 네트워크**에 연결되어 있어야 합니다
2. ✅ 컴퓨터의 IP 주소: **192.168.219.103**

## 실행 방법

### 1단계: Expo 웹 서버 시작 (LAN 모드)

터미널에서 다음 명령어를 실행하세요:

```bash
npx expo start --web --host lan
```

또는

```bash
npx expo start --web
```

그 다음 터미널에서 `w` 키를 누르면 웹 서버가 시작됩니다.

### 2단계: 터미널에서 표시된 주소 확인

웹 서버가 시작되면 터미널에 다음과 같은 주소들이 표시됩니다:

```
Web is waiting on http://192.168.219.103:8081
```

또는

```
Metro waiting on exp://192.168.219.103:8081
Web is waiting on http://192.168.219.103:8081
```

### 3단계: 핸드폰 브라우저에서 접속

1. **핸드폰의 Wi-Fi를 컴퓨터와 같은 네트워크로 연결**하세요
2. **핸드폰 브라우저**(Chrome, Safari 등)를 열고 다음 주소로 접속:
   ```
   http://192.168.219.103:8081
   ```
3. 또는 터미널에 표시된 정확한 주소를 입력하세요

## 문제 해결

### 핸드폰에서 접속이 안 되는 경우

#### 1. 방화벽 확인
Windows 방화벽이 8081 포트를 차단하고 있을 수 있습니다.

**해결 방법:**
- Windows 설정 > 네트워크 및 보안 > Windows Defender 방화벽
- "앱이 방화벽을 통과하도록 허용"에서 Node.js 허용 확인
- 또는 임시로 방화벽을 끄고 테스트

#### 2. 네트워크 확인
- 컴퓨터와 핸드폰이 **정확히 같은 Wi-Fi 네트워크**에 연결되어 있는지 확인
- Wi-Fi 이름(SSID)이 동일한지 확인

#### 3. IP 주소 재확인
IP 주소가 변경되었을 수 있으므로 다시 확인:

```bash
ipconfig | findstr /i "IPv4"
```

#### 4. 다른 포트 사용
8081 포트가 사용 중이면 다른 포트 사용:

```bash
npx expo start --web --port 8082
```

그리고 핸드폰에서 `http://192.168.219.103:8082` 접속

### 연결은 되는데 화면이 안 보이는 경우

1. 브라우저 캐시 삭제 후 재시도
2. 브라우저 개발자 도구(F12)에서 콘솔 에러 확인
3. 다른 브라우저로 시도 (Chrome, Safari, Firefox)

## 빠른 명령어 요약

```bash
# 1. 웹 서버 시작 (LAN 모드)
npx expo start --web --host lan

# 2. 터미널에서 'w' 키 누르기

# 3. 핸드폰 브라우저에서 접속
# http://192.168.219.103:8081
```

## 참고사항

- 웹에서 실행하면 네이티브 기능(카메라, 푸시 알림 등)이 제한될 수 있습니다
- 백엔드 API 서버도 같은 네트워크에서 실행 중이어야 정상 작동합니다
- 개발 모드에서는 핫 리로드가 작동하지 않을 수 있습니다 (새로고침 필요)
